玩转视口单位

运用视口单位实现响应式排版已蔚然成风——即根据当前视口尺寸自动调节字号大小。但直接使用基础视口单位设置字号会产生一个有趣（且危险）的现象：如您所见，字体缩放速度极快，在极小视口区间内就能完成从难以辨识的微缩字号到超大尺寸的剧烈变化。

这种直接缩放的方式在日常使用中显然过于夸张。我们需要更精细的调节方案——既要设定最小和最大阈值，又要能精准控制增长幅度。这时 calc() 就派上了用场。我们可以将相对稳定的单位（例如 16px ）作为基准尺寸，再辅以小幅度的视口自适应调整（ 0.5vw ），最后由浏览器完成计算： calc(16px + 0.5vw)

通过调整基础字号与视口相对值之间的比例关系，可以控制文本缩放幅度的强弱。对标题采用较大的视口单位值，即可使其比正文文本更快放大。这样既能在大屏幕上实现灵活多变的字体层级，又能在移动端保持字号克制——全程无需使用媒体查询。此技巧同样适用于行高设置，让行间距能够以独立于字号的速率进行自适应调整。
